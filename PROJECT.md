# Insurance CRM — Project Tracker

AI-powered CRM for an insurance agent (Life/Term + Health), built local-first on Windows.

## Decisions locked in
- **Hosting:** Local-first on the agent's PC (deploy option later).
- **WhatsApp:** Option A — unofficial QR link (whatsapp-web.js), send from own number, with safeguards.
- **Users:** Single agent now; schema kept team-ready.
- **Lines:** Life / Term / Health.
- **Languages:** English + Telugu (message templates & AI drafting).
- **First AI feature:** Daily Briefing.

## Tech stack
- **Next.js** (React + API in one app) · **TypeScript** · **Tailwind CSS**
- **SQLite** (single local file) via **Prisma**
- **AES-256-GCM** encryption for Aadhaar/PAN + uploaded documents (key in `.env`)
- **whatsapp-web.js** (Phase 3) · **Anthropic SDK / Claude** (Phase 4)

## How to run
**For the agent (no terminal):** double-click **`Start CRM.cmd`** (or run **`Create Desktop Shortcut.cmd`** once, then use the Desktop icon). It builds on first run, starts the production server, and opens http://localhost:3000; keep the window open while using the app, close it to quit. **`Update CRM.cmd`** rebuilds after code changes. See **`START HERE.txt`**. *(Verified: prod `next start` boots in <1s, AI key + WhatsApp session both work in production; WhatsApp reconnects from `.wwebjs_auth` with no QR after one Connect click.)*

**For development:**
```bash
npm install          # one-time, install dependencies
npm run setup        # one-time, create DB + load demo data
npm run dev          # dev server → http://localhost:3000  (slower; clobbers the prod build, so re-run Update CRM after)
npm run build && npm run start   # production (what the launcher uses)
```
Useful: `npm run db:studio` (browse the DB), `npm run db:reset` (wipe + reseed). Note: `next dev` and `next start` share `.next`, so the launcher rebuilds whenever a production `BUILD_ID` is missing.

## User guide
Illustrated end-to-end walkthrough for the agent: **`docs/GUIDE.html`** (double-click → browser, with screenshots) and **`docs/GUIDE.md`** (same content, for devs/GitHub). Covers every screen, the everyday flows (morning routine, one-tap AI renewal reminder, bulk import, upload-&-scan, lead→client), and a **data-flow diagram** (local SQLite + encrypted `/storage`; only non-sensitive facts to Claude, never Aadhaar/PAN; WhatsApp from own number). Screenshots auto-captured by `scripts/screenshots.mjs` (puppeteer against the running app → `docs/screenshots/`); re-run after UI changes.

## Security notes
- Aadhaar & PAN stored encrypted; masked on screen (last 4), reveal on click.
- `.env`, `dev.db`, and `/storage` are git-ignored — they hold secrets/client data.
- Nothing sensitive leaves the machine; AI calls (Phase 4) go only to Anthropic on the agent's key.
